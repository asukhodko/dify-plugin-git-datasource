## Полезные ссылки и примеры

При разработке и тестировании рекомендуем ознакомиться со следующими официальными материалами и примерами, которые послужат ориентирами:

- **Data Source Plugin – Dify Docs:** Официальная документация по разработке плагинов-источников данных (включая веб-краулеры). Содержит обзор типов плагинов и пример реализации метода `_get_website_crawl`[docs.dify.ai](https://docs.dify.ai/en/develop-plugin/dev-guides-and-walkthroughs/datasource-plugin#:~:text=In%20the%20main%20logic%20code,completed)[docs.dify.ai](https://docs.dify.ai/en/develop-plugin/dev-guides-and-walkthroughs/datasource-plugin#:~:text=crawl_res.status%20%3D%20,%29). Полный текст: _“Develop a Data Source Plugin”_ на docs.dify.ai.
    
- **Структура провайдера и manifest:** В документации Dify приведён пример конфигурации провайдера с различными `provider_type` (online_drive, online_document, website_crawl)[docs.dify.ai](https://docs.dify.ai/en/develop-plugin/dev-guides-and-walkthroughs/datasource-plugin#:~:text=Copy). Убедитесь, что в вашем `provider.yaml` указан `website_crawl` и прописаны все необходимые параметры (datasource URL, ключи и т.д.).
    
- **Пример выходных данных Web Crawler:** Документация описывает, какие поля ожидаются в выходном schema веб-краулера (content, title, description)[docs.dify.ai](https://docs.dify.ai/en/develop-plugin/dev-guides-and-walkthroughs/datasource-plugin#:~:text=type%3A%20string%20description%3A%20the%20content,the%20description%20of%20the%20website), а также механизм пакетной выдачи с статусами processing/completed[docs.dify.ai](https://docs.dify.ai/en/develop-plugin/dev-guides-and-walkthroughs/datasource-plugin#:~:text=In%20the%20main%20logic%20code,completed). Это напрямую применимо к нашему Git-плагину.
    
- **Режим Sync from Website – руководство пользователя Dify:** В разделе пользовательского гайда описан процесс импорта данных с веб-сайта[docs.dify.ai](https://docs.dify.ai/versions/3-0-x/en/user-guide/knowledge-base/create-knowledge-and-upload-documents/import-content-data/sync-from-website#:~:text=The%20knowledge%20base%20supports%20crawling,it%20into%20the%20knowledge%20base). Там же приводится пример использования Firecrawl и Jina Reader, что подтверждает: Dify может брать **несколько страниц за раз** и после парсинга сохранять их как отдельные документы. Наш плагин будет аналогичным по поведению, только для Git-страниц. Смотри “Import Data from Website” в Dify Docs[docs.dify.ai](https://docs.dify.ai/versions/3-0-x/en/user-guide/knowledge-base/create-knowledge-and-upload-documents/import-content-data/sync-from-website#:~:text=2).
    
- **Dify Marketplace и примеры плагинов:** На маркетплейсе Dify есть **GitHub Datasource Plugin** (организация langgenius, ID github). Хотя его реализация отличается (работает через OAuth и GitHub API), описание может быть полезно. Также обратите внимание на **Crawl4AI** и **Firecrawl** – эти плагины доступны в Marketplace и их исходники частично открыты. Например, Firecrawl (версия 0.0.3) перечислен в официальном репозитории плагинов[github.com](https://github.com/svcvit/dify_plugin_collection#:~:text=hjlarry%2Fdatabase%20database%20%E6%AD%A4%E5%B7%A5%E5%85%B7%E5%8F%AF%E4%BB%A5%E7%94%A8%E6%9D%A5%E8%BF%9E%E6%8E%A5%E7%8E%B0%E6%9C%89%E6%95%B0%E6%8D%AE%E5%BA%93%EF%BC%8C%E5%B9%B6%E6%89%A7%E8%A1%8CSQL%E8%AF%AD%E5%8F%A5%E5%92%8C%E8%87%AA%E7%84%B6%E8%AF%AD%E8%A8%80%E8%BD%ACSQL%E3%80%82%20hjlarry_database_0,langgenius%2Fwikipedia%20%E7%BB%B4%E5%9F%BA%E7%99%BE%E7%A7%91%20%E7%BB%B4%E5%9F%BA%E7%99%BE%E7%A7%91%E6%98%AF%E4%B8%80%E4%B8%AA%E7%94%B1%E5%85%A8%E4%B8%96%E7%95%8C%E7%9A%84%E5%BF%97%E6%84%BF%E8%80%85%E5%88%9B%E5%BB%BA%E5%92%8C%E7%BC%96%E8%BE%91%E7%9A%84%E5%85%8D%E8%B4%B9%E5%9C%A8%E7%BA%BF%E7%99%BE%E7%A7%91%E5%85%A8%E4%B9%A6%E3%80%82%20%20161)[github.com](https://github.com/svcvit/dify_plugin_collection#:~:text=langgenius%2Ffirecrawl%20Firecrawl%20Firecrawl%20API%20%E9%9B%86%E6%88%90%EF%BC%8C%E7%94%A8%E4%BA%8E%E7%BD%91%E9%A1%B5%E7%88%AC%E5%8F%96%E5%92%8C%E6%95%B0%E6%8D%AE%E6%8A%93%E5%8F%96%E3%80%82,langgenius%2Fwecom%20%E4%BC%81%E4%B8%9A%E5%BE%AE%E4%BF%A1%20%E4%BC%81%E4%B8%9A%E5%BE%AE%E4%BF%A1%E7%BE%A4%E6%9C%BA%E5%99%A8%E4%BA%BA%20%20162) и был интегрирован в Dify v0.6.11 как решение для веб-краулинга[dify.ai](https://dify.ai/blog/dify-ai-blog-integrated-with-firecrawl#:~:text=Dify.AI%20x%20Firecrawl%3A%20A%20Top,for%20Web%20Data%20Knowledge%20Base)[dify.ai](https://dify.ai/blog/dify-ai-blog-integrated-with-firecrawl#:~:text=Firecrawl%20can%20turn%20websites%20into,your%20RAG%20application%20on%20Dify).
    
- **Официальный репозиторий плагинов (GitHub):** dify-official-plugins – содержит код всех плагинов от команды Dify[github.com](https://github.com/langgenius/dify-official-plugins#:~:text=Dify%27s%20models%20and%20tools%20were,users%20to%20explore%20and%20use). Вы можете найти там реализацию Notion (online_document) или Google Drive (online_drive) для сравнения, а также убедиться, как оформлены README и Privacy. При публикации своего плагина, следуйте тем же стандартам.
    
- **Сообщество и примеры:** Сторонние разработчики выкладывали свои плагины для Dify на GitHub. Например, репозиторий **dify_plugin_collection** собрал многие `.difypkg` из Marketplace[github.com](https://github.com/svcvit/dify_plugin_collection#:~:text=Dify%20Plugin%20List)[github.com](https://github.com/svcvit/dify_plugin_collection#:~:text=%E8%BF%99%E6%98%AF%20DIFY%20%E7%9A%84%E5%AE%98%E6%96%B9%E5%B8%82%E5%9C%BA%20https%3A%2F%2Fmarketplace) – оттуда можно извлечь исходники, распаковав `.difypkg` (это zip-архив). В частности, можно попробовать достать код `langgenius/github_0.0.5` или `agimaster/crawl4ai` из этой коллекции, чтобы увидеть, как они устроены.
    
- **Чеклист разработки плагина:** Dify Docs предлагает _Plugin Development Cheatsheet_, где перечислены шаги создания и отладки плагина[docs.dify.ai](https://docs.dify.ai/en/develop-plugin/getting-started/getting-started-dify-plugin#:~:text=Dify%20plugins%20are%20modular%20components,built%20AI%20applications)[docs.dify.ai](https://docs.dify.ai/en/develop-plugin/getting-started/getting-started-dify-plugin#:~:text=Additional%20Resources). Полезно пробежать, чтобы не забыть про `minimum_dify_version`, правильное именование классов, зависимости (`requirements.txt` должен включать `dify-plugin>=0.5.0,<0.6.0` для data source plugin[docs.dify.ai](https://docs.dify.ai/en/develop-plugin/dev-guides-and-walkthroughs/datasource-plugin#:~:text=,source%20plugin%20development%20as%20follows)) и т.д.
